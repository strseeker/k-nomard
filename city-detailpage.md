## 1. 기능 분석

### 핵심 목표
사용자가 도시 카드 클릭 시 `/cities/[id]` 상세 페이지로 이동하여 해당 도시의 모든 정보(예산, 환경, 계절, 태그, 좋아요)를 한 눈에 확인할 수 있다.

### 범위 정의
| 포함 (In Scope) | 제외 (Out of Scope) |
|---|---|
| 도시 상세 페이지 라우트 생성 | 도시 데이터 Supabase DB 마이그레이션 |
| 도시 기본 정보 표시 (budget, region, environment, bestSeason, tags) | 리뷰 작성/수정 기능 |
| 좋아요/싫어요 인터랙션 | 지도 API 연동 |
| CityCard → 상세 페이지 링크 연결 | 도시 간 비교 기능 |
| 404 처리 (존재하지 않는 id) | 이미지 갤러리 |

### 사용자 시나리오

**시나리오 1 — 도시 탐색 후 상세 확인**
- **사용자:** 워케이션을 계획 중인 디지털 노마드
- **행동:** 홈 화면 CityGrid에서 "제주시" 카드를 클릭한다
- **기대 결과:** `/cities/jeju` 로 이동, 제주시의 예산·환경·태그 등 전체 정보를 확인한다

**시나리오 2 — 직접 URL 접근**
- **사용자:** 링크를 공유 받은 사용자
- **행동:** `/cities/busan` URL을 직접 입력하거나 공유 링크로 접근한다
- **기대 결과:** 부산 상세 페이지가 올바르게 렌더링된다

**시나리오 3 — 잘못된 도시 ID 접근**
- **사용자:** 잘못된 URL로 접근한 사용자
- **행동:** `/cities/nonexistent` 접근
- **기대 결과:** 404 페이지 또는 "도시를 찾을 수 없습니다" 안내 화면 표시

## 2. 아키텍처 분해

### 프런트엔드 구성
- **페이지/라우트:**
  - `src/app/(main)/cities/[id]/page.tsx` — 신규 생성 (Server Component)
- **컴포넌트:**
  - `src/components/city-detail/CityDetailHero.tsx` — 도시 히어로 이미지 + 뱃지 영역 (신규)
  - `src/components/city-detail/CityInfoGrid.tsx` — 예산/지역/환경/계절 정보 그리드 (신규)
  - `src/components/city-detail/CityTagList.tsx` — 태그 목록 표시 (신규)
  - `src/components/city-detail/CityVoteBar.tsx` — 좋아요/싫어요 인터랙션 (신규, Client Component)
  - `src/components/home/CityCard.tsx` — 링크(`<Link href>`) 추가 (수정)
- **상태 관리:** 좋아요/싫어요만 `useState` (Client Component), 나머지는 Server Component
- **UI 상호작용:** 카드 클릭 → 라우팅, 좋아요/싫어요 토글

### 백엔드 구성
- **API / Server Actions:** 없음 (현재 정적 데이터 기반)
- **데이터베이스:** 불필요 (`src/data/cities.ts` 에서 `id`로 직접 조회)
- **인증/권한:** 불필요 (공개 페이지)
- **비즈니스 로직:** `cities.find(c => c.id === params.id)` → 없으면 `notFound()` 호출

### 외부 연동
- **서드파티:** 없음
- **환경변수:** 추가 불필요
- **의존 패키지:** 추가 불필요 (lucide-react, shadcn/ui 기존 사용 중)
