# breakdown 커맨드

## 사용방법

```
/breakdown <해결해야 할 문제 또는 기능 설명>
```

**예시:**
```
/breakdown 사용자 리뷰 작성 기능 구현
/breakdown 로그인/회원가입 페이지 및 Supabase Auth 연동
/breakdown 도시 검색 및 필터링 기능 추가
```

## 2. 아키텍처 분해

### 프런트엔드 구성
- **페이지/라우트:** 새로 추가되거나 수정되는 경로
- **컴포넌트:** 신규 생성 / 수정 컴포넌트 목록
- **상태 관리:** 클라이언트 상태, 서버 상태 처리 방식
- **UI 상호작용:** 주요 이벤트 및 사용자 흐름

### 백엔드 구성
- **API 엔드포인트 / Server Actions:** 필요한 서버 로직
- **데이터베이스:** 테이블, 쿼리, 마이그레이션 필요 여부
- **인증/권한:** 접근 제어 필요 여부
- **비즈니스 로직:** 핵심 처리 로직 요약

### 외부 연동
- **서드파티 서비스:** 사용하는 외부 API, SDK
- **환경변수:** 추가 필요한 설정값
- **의존 패키지:** 신규 설치 필요 라이브러리
