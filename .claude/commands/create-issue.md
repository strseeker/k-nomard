# GitHub 이슈 생성 커맨드

## 사용방법

```
/create-issue <생성할 이슈 설명>
```

**예시:**
```
/create-issue 도시 상세 페이지 구현
/create-issue 로그인 후 리다이렉트가 동작하지 않는 버그 수정
/create-issue 리뷰 작성 폼 UI 개선
```

---

## 프롬프트

당신은 K·NOMADS 프로젝트의 GitHub 이슈를 작성하는 전문가입니다.
아래 요청을 분석하여 명확하고 실행 가능한 GitHub 이슈를 생성하세요.

**요청:** $ARGUMENTS

---

다음 단계를 순서대로 수행하세요:

### 1단계: 이슈 분류 결정

요청 내용을 분석하여 아래 중 하나로 분류하세요:
- `feat`: 신규 기능
- `bug`: 버그 수정
- `chore`: 리팩토링, 설정, 의존성 업데이트
- `docs`: 문서
- `design`: UI/UX 개선

### 2단계: 이슈 내용 작성

아래 형식에 맞게 이슈 제목과 본문을 작성하세요.

**제목 형식:** `[타입] 간결한 이슈 제목`

**본문 형식:**

```markdown
## 개요
> 이 이슈를 한 문장으로 설명하세요.

## 배경 / 문제 상황
(왜 이 이슈가 필요한지, 현재 어떤 문제가 있는지 설명)

## 목표
- [ ] 달성해야 할 구체적인 결과 1
- [ ] 달성해야 할 구체적인 결과 2

## 구현 범위
**포함:**
-

**제외:**
-

## 참고 사항
(관련 파일 경로, 기술 결정, 참고 링크 등)

## 완료 조건 (Definition of Done)
- [ ] 조건 1
- [ ] 조건 2
- [ ] 코드 리뷰 완료
```

### 3단계: 라벨 결정

아래 라벨 중 적합한 것을 선택하세요 (없으면 생략):
- `enhancement` (신규 기능)
- `bug` (버그)
- `good first issue` (입문자용)
- `help wanted` (도움 필요)
- `documentation` (문서)

### 4단계: GitHub 이슈 생성

위에서 작성한 내용으로 아래 명령어를 실행하세요:

```bash
gh issue create \
  --title "<작성한 제목>" \
  --body "$(cat <<'EOF'
<작성한 본문>
EOF
)" \
  --label "<라벨>"
```

이슈 생성 후 생성된 이슈 URL을 사용자에게 알려주세요.
